# ZZU_North_Card 校园卡灯板项目

ZZU_North_Card 是一个基于 ESP8266 芯片开发的智能校园卡灯板项目。它不仅是一款精美的装饰品，更集成了丰富的灯光效果、低功耗管理以及便捷的 Web 控制界面，旨在为你的校园卡增添科技感与个性化色彩。

## 项目特性/功能介绍

- **多种灯光模式**：支持常亮、脉冲、呼吸、彩虹渐变等多种预设灯光效果。
- **自定义色彩**：内置色轮模式，用户可以通过手机 Web 界面自由选取任意颜色。
- **双路独立控制**：由ESP8266芯片负责RGB灯珠的WiFi网页与物理按键控制；此外，板载独立的**SGL8022W触摸芯片**直接控制另外两路灯丝，实现“一板双控”。
- **智能电源管理**：
  - **自动休眠**：当处于“休眠态”时，无任何操作30秒后，系统将自动关闭WiFi模块进入低功耗模式，显著降低待机功耗。
  - **按键唤醒**：通过侧边物理按键即可轻松唤醒系统。
- **便捷控制交互**：
  - **Web 控制台**：无需安装 App，连接热点即可通过手机浏览器调整颜色、模式和亮度。
  - **物理按键**：支持短按切换模式，长按无级调光。
- **OTA 在线升级**：支持通过 Web 界面无线上传固件（.bin 格式），方便功能迭代与更新。

## 硬件要求和连接说明

### 硬件清单
- **核心控制器**：ESP8266 (ESP-12F 模块)
- **发光组件**：4颗高亮5050共阳RGB灯珠、2路独立灯丝
- **触摸控制**：SGL8022W 单通道触摸芯片（用于独立控制灯丝）
- **驱动电路**：3个 AO3400 N沟道MOS管（用于RGB灯珠）
- **按键**：一个物理轻触按键（yd-3414）
- **电源**：3.7V 锂电池及充放一体模块 (5V升压输出)

### 引脚连接

#### ESP8266 部分
| 组件 | ESP8266 引脚 (GPIO) | 说明 |
| :--- | :--- | :--- |
| **KEY_PIN** | GPIO 4 | 控制按键 |
| **LED_R** | GPIO 5 | RGB 灯红光控制 |
| **LED_G** | GPIO 12 | RGB 灯绿光控制 |
| **LED_B** | GPIO 13 | RGB 灯蓝光控制 |
| **LED_BUILTIN**| GPIO 2 | 板载提示灯 |

#### 独立触摸灯丝控制电路 (SGL8022W)
| 组件 | 连接点 | 说明 |
| :--- | :--- | :--- |
| **VDD** | 3.3V | 芯片供电 |
| **TI (触摸输入)** | 校徽覆铜触摸盘 | 触摸感应输入端 |
| **SO (控制输出)** | 灯丝驱动端 | 输出PWM信号控制灯丝亮度 |
| **OPT1/OPT2** | 根据需求配置 | 用于选择调光模式（如带亮度记忆） |
| **VSS** | GND | 芯片地 |

## 软件安装和配置指南

### 环境准备
1. 安装 [Arduino IDE](https://www.arduino.cc/en/software)。
2. 在 Arduino IDE 中添加 ESP8266 开发板支持：
   - 进入 `首选项` -> `附加开发板管理器网址`，添加 `http://arduino.esp8266.com/stable/package_esp8266com_index.json`。
   - 在 `开发板管理器` 中搜索并安装 `esp8266`。

### 固件烧录
1. 克隆本项目代码到本地。
2. 使用 Arduino IDE 打开项目中的 `.ino` 文件。
3. 项目启动后，ESP8266会创建一个**动态热点**。热点名称以 `CampusLight_` 为前缀，后四位来自设备MAC地址（例如 `CampusLight_9EC3`）。如果需要修改，可以更改代码开头的 `ap_ssid` 和 `ap_password` 宏定义。
4. 将 ESP8266 通过 USB 转 TTL 模块连接至电脑，注意烧录时需将GPIO0引脚临时接地。在开发板菜单中选择 `NodeMCU 1.0 (ESP-12E Module)` 及正确的COM口。
5. 点击“上传”按钮进行烧录。

## 使用方法和操作说明

### 网络连接
1. 开启设备，设备将自动创建名为 **`CampusLight_XXXX`** 的 WiFi 热点（XXXX为设备MAC地址后四位）。
2. 使用手机连接该热点（默认密码：`12345678`）。
3. 大部分手机会自动弹出登录页面（强制门户），若未弹出，请在浏览器访问 `192.168.4.1`。

### 界面控制
- **模式切换**：点击页面上的“休眠”、“恒光”、“色谱”、“脉冲”、“呼吸”、“彩虹”等按钮进行切换。
- **亮度调节**：拖动滑块可实时改变灯板亮度。
- **颜色选取**：在色轮上点击或拖动以选择自定义颜色。
- **固件升级**：点击页面底部的红色“固件升级 (OTA)”按钮，即可进入OTA升级页面。
- **独立灯丝控制**：校徽部分的灯丝由独立的触摸芯片控制，短按触摸盘即可开关，长按可无级调光。

### 按键操作
- **短按**：在预设的几种模式间循环切换。
- **长按**：无级调节当前灯光的亮度。
- **休眠唤醒**：在设备休眠时，按下按键即可唤醒进入工作状态。

## 项目架构和技术细节

- **后端实现**：基于 `ESP8266WebServer` 实现轻量级 HTTP 服务器，处理来自前端的控制指令和状态请求。
- **前端技术**：采用原生的 HTML5、CSS3 和 JavaScript 开发。利用 `Canvas` 实现色轮取色，通过 `fetch` API 与后端进行非阻塞通信。
- **核心逻辑**：
  - **主控逻辑**：负责系统状态管理、模式切换和非阻塞式动画效果更新。
  - **硬件驱动层**：封装了底层的 PWM 调光控制和按键检测算法（包含软件消抖）。
  - **网络服务层**：响应 `/mode`, `/color`, `/bright`, `/status`, `/colorwheel` 等 API 请求，并集成OTA升级服务。
  - **独立触摸电路**：另一路灯光通过独立的 **SGL8022W 触摸芯片**控制，无需编程，仅通过外围电阻电容即可实现短按开关、长按调光、亮度记忆等功能，与ESP8266系统互不干扰。
- **能耗优化**：当系统处于“休眠态”且30秒无操作后，ESP8266将通过软件指令主动关闭WiFi射频模块，并降低CPU运行频率，以降低待机功耗。同时，用户可通过外置拨动开关或物理按键重新唤醒系统。

## 故障排除和常见问题

1. **连接 WiFi 后不弹出控制页面？**
   - 尝试在浏览器手动输入 `192.168.4.1`。
2. **灯光颜色不准确？**
   - 请检查项目源码中LED_R、LED_G、LED_B的引脚定义是否与实际硬件连接一致。
3. **按键无法唤醒？**
   - 确保按键连接到了正确的 GPIO 引脚，并且连接了适当的上拉/下拉电阻（代码中默认开启了内部上拉）。
4. **校徽触摸灯丝不响应？**
   - 请检查SGL8022W芯片的外围电路，特别是触摸盘引脚的22pF灵敏度电容是否焊接牢固。确认芯片的OPT1和OPT2引脚配置是否符合预期功能模式。
5. **OTA升级失败？**
   - 请确保在升级过程中电源稳定，不要断开。如果失败，可能需要重新通过串口烧录一次完整的固件以恢复系统。
